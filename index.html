Data Cleaning

When cleaning the data, there were numerous errors which were found and cleaned. These can be divided into a number of categories and subcategories. OpenRefine was used for the data cleaning with occasional viewings in Excel. The list below shows the type of errors, the columns affected (where applicable) and the solution.

  • Date Validation
    • Start Date
    • Completion Date
    • Planned Completion Date
    • Actual Completion Date
    • Combined Updated time
    Solution: Each of these dates was selected and a common transform was applied, changing them to a date data type. Combined Updated Time was created from Updated Time and Date to form exact time + date.
  • Multiple Representations
    • Agency Name
    • Agency ID
    • Project Description
    Solution: For each of these, a text-facet was applied and the data was grouped together. Different algorithms were checked to test similarity and then converted to the most common name.
  • Summation Records
    • Solution:Each row called 'Total' was starred and removed.
  • Duplicate Records
    • Duplicate Project ID records
    Solution: ProjectID was the column with a uniqe identifier. After performing a text facet, rows with the same were looked at and duplicate records deleted.
  • Mixed use of Numerical scales
    • Lifecycle Cost
    Solution: This was one of the more challenging errors to fix. This value was compared to the Projected Cost and Actual cost and brought as close to them as posisble if it seemed too much. For example, if Project and Actual cost was 5, and the lifecycle cost was 5000,it was assumed that an error is present. Similarly, if the projected cost is 5654 and actual cost is 256454, then it was assumed that 2 was added by an accident. A few more were more ambigious and the best judgement was used.
  • Unorganised Data
    • Solution: Some of rows had an extra column. They were studied to see where the problem was (generally an extra value in Project Name and everything shifted forward). Excel was used to quickly copy the rows to the correct place.
  • Spellling Errors
    • Agency
    • Project Description
    Solution:This was also mostly solved through text facets to the best of knowledge.

Visualisations

Treemap Visualisation

Description

The treemap shows how much money is being spent by the different agencies in the US. For each agency, money division over the different investment titles is also shown, which is then showed by each project.

Audience

The intended audience is the general public who wish to see how their tax dollars are being invested/used. This is quite simple and easy to understand.

Interaction

The colouring of the treemap is done individually for the different values at each level. Therefore, each department is a different colour. Moreover, when a user hovers over a block, the information of it and the money being spent is easily shown, which is helpful especially when the user cannot read the smaller boxes.

Scatter Plot Visualisation

Description

This visualisation is a scatter plot of the data of the Duration of project (in days) against the projected cost($M) of the project

Audience

This would be for people who want to see how the cost of the project might co-relate with the amount of money being spent on it. They might also be able to see any weird projects which seem to finish quickly but cost a lot of money and may want to find out more.

Interaction

The graph has a legend which shows the different agencies to allow the user to distinguish between projects. Moreover, the map is zoomable, which means that more precision can be seen.
